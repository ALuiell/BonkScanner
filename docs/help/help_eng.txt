BonkScanner - Quick Help
========================

Quick Start
-----------
1. Launch Megabonk and wait until the target scene is fully loaded.
2. Open BonkScanner.
3. Choose `Templates` or `Scores` on the left side.
4. Configure the active templates or score tiers.
5. Press `Start`.
6. Use the scan hotkey in-game to arm or pause the loop.
7. When a map matches your conditions, BonkScanner stops and writes the result to `Logs`.

Search Modes
------------
`Templates`
- Strict search based on exact conditions.
- Best when you want specific values such as `Shady Guy`, `Moais`, `Microwaves`, `Boss Curses`, or `S+M`.

`Scores`
- Flexible search based on overall map value.
- Better when you want generally strong maps instead of one exact combination.

Important:
- The active left-side tab decides which mode is currently used.
- You can change active templates or score tiers while a search loop is already running. BonkScanner applies the new active filters live.

Main UI Areas
-------------
Left side:
- `Templates` - strict rule-based filtering with active template selection.
- `Scores` - weighted map evaluation with active score tier selection.

Right side:
- `Logs` - scanner activity, warnings, wait states, and match results.
- `Session Stats` - session time, reroll count, RPM, best and worst maps, and averages per target.
- `Live Stats` - current run stats, items, weapons, tomes, and recording controls.
- `Recordings` - saved recordings viewer with timeline, rename, delete, and cleanup actions.
- `Compare Runs` - side-by-side comparison of two saved runs with synced in-game time and a central difference panel.
- `OBS Overlay` - local browser-source overlay for stream layouts.
- `Twitch Bot` - built-in Twitch bot for chat integration.
- `In-Game Overlay` - transparent in-game window overlay.

Reroll and Restart
------------------
- Standard keyboard restart is still the default and simplest option.
- If rerolls feel unstable, first try increasing `Min Reroll Delay (s)` or `Reset Hold Duration (s)` a little.
- BonkScanner waits for a safe map-ready and memory-read state, so short pauses between restart and evaluation are normal.

Live Stats
----------
The `Live Stats` tab shows:
- current player stats;
- passive items with rarity highlighting and sorting;
- `Average chests/min`;
- `In-Game Time`;
- `Mob Kills`;
- `Level`;
- `Stage Summary`;
- `Powerups`;
- `Banishes`.

Additional tabs inside `Live Stats`:
- `Stats` - main character stats;
- `Weapons` - current weapons, their levels, and upgraded weapon stats;
- `Tomes` - current tomes, their levels, and active effects;
- `Chaos` - Chaos Tome level and total stat bonuses recovered from tracked rolls;
- `Damage Sources` - detailed statistics for each damage source/weapon.

The `Powerups` card tracks active powerups (Rage, Shield, Stonks, Clock) and shows their pickup and expiration timestamps.

If some sections temporarily say that data is unavailable, that is not always an error. During loading, some memory pointers may not be ready yet.

Recording and Recordings
------------------------
- Recording can be started from `Live Stats` or through the `Record Hotkey`.
- Snapshots are saved using the interval from `Snapshot Interval (s)`.
- Saved runs can later be reviewed in the `Recordings` tab.

Inside `Recordings`, you can:
- scrub the timeline;
- inspect stats, items, weapons, tomes, chaos, damage sources, stage summary, and banishes by snapshot;
- set a compare start and compare a later snapshot against it;
- review compare details such as gained, broken, and lost items between two snapshots;
- rename a recording;
- delete a recording;
- clean short recordings by snapshot count.

Recording behavior:
- normal stage transitions should not split a recording into a new file;
- when the app detects a truly new run, recording can automatically continue into a new file;
- if the current run seed disappears and stays unavailable, recording can stop automatically;
- older files remain compatible even if they do not contain newer fields such as `weapons`, `tomes`, or `banishes`.

Compare Runs
------------
- `Compare Runs` loads two saved recordings side by side as `Run A` and `Run B`.
- If no runs are selected yet, the recordings chooser opens automatically the first time you enter this tab.
- During that guided first selection, the chooser closes automatically after both runs are selected.
- Snapshots can be compared by matched `In-Game Time`, so both runs stay aligned to the closest moment.
- The center `Difference` panel shows overview deltas and optional sections for stats, stage summary, items, weapons, and tomes.
- `Compare Settings` lets you choose which stats appear in the difference view and which compare sections stay visible.

OBS Overlay
-----------
- `OBS Overlay` creates a small local web page that OBS can show as a Browser Source on top of gameplay.
- It runs only on the same PC. The server binds to `127.0.0.1`, which means local computer only, not the public internet.
- Turn on `OBS Overlay server`, then copy the `Local OBS URL` into an OBS Browser Source.
- The URL looks like `http://127.0.0.1:17845/overlay`: `127.0.0.1` is your own PC, `17845` is the port, and `/overlay` is the overlay page.
- **Visual Layout Editor**: Open the URL in any browser with `?edit=true` (e.g. `http://127.0.0.1:17845/overlay?edit=true`) to visually drag and drop widgets, resize the canvas resolution to match your stream HUD, and save layout changes instantly and automatically!
- **Fixed Canvas Size**: When absolute positioning is active, the overlay uses a customizable canvas size (default: 1920x1080). Make sure to set the identical Width and Height values in your OBS Browser Source settings.
- A port is the local number the browser source uses to reach the overlay server. If another app already uses that port, choose a different one and update the OBS Browser Source URL.
- Recording is not required. The overlay uses live stats reads and can work while `.jsonl` recording is off.
- Widget selection controls what appears on stream. Available widgets are `Stage Summary`, `Tracked Items`, `Stats`, `KPS`, and `Banishes`.
- **KPS Widget Settings**: For the KPS overlay widget, you can toggle its background opacity, headers, and customize the metrics shown (Current KPS, 60s Avg, 5m Avg, and Run Avg).
- Tracked items count selected item gains, such as first-map item luck. Search for an item, choose whether it should count `Map 1 only`, then add it to the tracked list.
- The Tracked Items widget uses its own custom list by default. Enable `Use Session Stats tracked items` to show the same counters as `Session Stats` without deleting the custom overlay list.
- The overlay page is transparent, so OBS should show only the compact text/widgets over the game instead of a large solid background.
- **Widget Scaling**: In Layout Editor mode, hover over any widget to reveal a scale toolbar. Use `+` / `-` buttons or type a custom percentage (e.g. `125`) directly to zoom individual widgets.
- **Widget Resizing**: Drag the bottom-right corner of any widget in Layout Editor to stretch its width and height. Stats and Banishes automatically reflow into multiple columns when widened.
- **Game Preview Background**: Layout Editor shows an actual game screenshot as background so you can align widgets precisely with the in-game HUD.
- **Widget-Specific URLs**: Each widget can be added as a separate OBS Browser Source using its own URL (e.g. `/overlay/stats`, `/overlay/banishes`, `/overlay/tracked_items`, `/overlay/stage_summary`, `/overlay/kps`).
- **Important: OBS Cache**: After updating BonkScanner or changing layout, right-click your Browser Source in OBS → Properties → click `Refresh cache of current page` → OK. Without this, OBS may show the old layout.


In-Game Overlay
---------------
- `In-Game Overlay` creates a transparent, floating window displayed directly on top of the game window.
- It displays live information during gameplay without needing a second monitor or an OBS Browser Source.
- Turn on `In-Game Overlay` in the settings tab, or enable `Auto-start overlay` to launch it automatically.
- **Layout Editor**: Press `F9` (default hotkey) or click `Edit Layout` in the app UI to toggle Layout Mode.
- In Layout Mode, you can drag widgets directly on top of the game window to align them. Press `Save Layout`, `Esc`, or `F9` again to exit and save.
- Widget scales (from `0.5` to `3.0`) can be adjusted in the `Widget Settings` dialog.
- Available overlay widgets:
  - `Scanner Status` — shows if the scanner is currently active.
  - `Recording Status` — shows if player stats recording is currently running.
  - `KPS` — shows kills per second metrics (with customizable sub-metrics like instant, 60s, 5m, or run averages).
  - `Active Powerups (Buffs)` — displays active powerups (Rage, Shield, Stonks, Clock) and their timers.
  - `Luck Rarity %` — shows the probability of item rarity rolls based on current Luck, with an optional progress bar.
- Note: The overlay will automatically hide when the game window loses focus, and reappear when it is focused.


Twitch Bot
----------
The `Twitch Bot` tab runs a built-in Twitch IRC chat bot for your channel.
This allows chat viewers to inquire about your current progress in real time!

How to configure:
1. Go to the `Twitch Bot` tab.
2. Click `Connect to Twitch`. A browser window will open for secure authorization via the official Twitch OAuth service.
3. Allow access, after which the status in the app will change to `Connected as [ChannelName]`.
4. Configure bot parameters (target channel, command access tier, cooldown delay, enabled commands, and auto-announcements).
5. Click `Start Bot`.

By default, `Target Channel` uses the authorized Twitch account. If you authorize a separate bot account, set `Target Channel` to the streamer channel where the bot should join and respond.
In `Command Settings`, `!session` uses `Session Stats` tracked items by default. Disable `Use Session Stats tracked items` to maintain a separate custom Twitch list.

Available chat commands:
- `!stats` / `!bonkstats` — displays character's current stats (damage, experience, luck, size, etc.).
- `!session` — displays current app session stats: resets, found seeds, seed rate, and configured tracked item counters.
- `!bans` / `!banishes` — list of banished items.
- `!disabled` — lists the highlighted/selected items that are globally disabled in the game lobby settings. (Note: due to game architecture, actual disabled items are only available in memory once a match starts. When configuring this via the bot UI in the main menu, use "Show all items" to select them beforehand.)
- `!items` — list of collected items (automatically sorted by rarity; if there are too many items, the bot compresses the list to fit Twitch's limits).
- `!weapons` — your weapons and only their active/upgraded stats.
- `!tomes` — your tomes with their stat values (Chaos Tome shows level only).
- `!chaos` / `!chaostome` — displays tracked Chaos Tome level and stat roll totals.
- `!kps` — displays current kills per second (KPS) and short/long averages (60s, 5m, run averages).
- `!stages` — detailed stage summary of kills and elapsed time for stages 1-4.
- `!powerups` — displays active powerups (Rage, Shield, Stonks, Clock) with their pickup and expiration timestamps.
- `!chests` / `!chest` — displays the number of chests opened per stage, keys count, and free chest openings (Free chest includes naturally free chests + chests opened for free using keys).
- `!presets` / `!preset` — displays active reroller templates or score tiers and weights.
- `!bonkhelp` — lists all active scanner bot commands (aliases: `!bonkcmds`, `!bonkcommands`, `!bhelp`).
- `!scanner` — general info about the BonkScanner app and download link.

Announce Stage Transitions:
If enabled, the bot automatically sends a summary to Twitch chat when transitioning between stages, e.g.:
🚩 Stage 1 completed! Kills: 1,042 | Time: 05:00. Moving to Stage 2! 🚩


Hotkeys
-------
`Settings` currently includes:
- `Scan Hotkey`
- `Reset Hotkey`
- `Record Hotkey`
- `In-Game Overlay Edit Hotkey`
- `Auto-start recording` - automatically starts player-stats recording when a live run is detected.
- `Auto-start overlay` - automatically starts the transparent overlay when a live run is detected.
- `Show OBS reminder on Start Scanner`

Important:
- `Reset Hotkey` is the community restart path. BonkScanner sends this key to restart runs.
- Some hotkeys on Windows may require Administrator privileges.
- If a hotkey does nothing, first make sure the game is open, focused when needed, and BonkScanner has attached cleanly.

Settings and Updates
--------------------
Main settings:
- `Scan Hotkey` - starts or pauses the scan loop while the app is running.
- `Reset Hotkey` - keyboard restart hotkey.
- `Record Hotkey` - starts or stops live stats recording.
- `In-Game Overlay Edit Hotkey` - enters or exits overlay Layout Mode.
- `Min Reroll Delay (s)` - minimum delay before the app evaluates the next reroll result.
- `Reset Hold Duration (s)` - keyboard reset hold time in the app. This setting also syncs the game's `quick_reset_time` value. If you change it, restart the game so the in-game restart timing fully picks up the new value.
- `Snapshot Interval (s)` - how often recording saves a new snapshot.
- `Check for Updates` - checks for a newer packaged build when that updater flow is available.

Other notes:
- Main user settings are stored in `config.json`.

If Something Looks Wrong
------------------------
- Start with the `Logs` tab. It usually explains why the app is waiting, warning, or refusing an action.
- Brief memory-read failures during loading screens or unstable game states are expected.
- Filter changes now apply live, so you usually do not need to restart the search loop just to update active templates or score tiers.
- If something looks stuck after major settings changes, the fastest recovery step is often restarting the loop with `Stop` -> `Start`.
