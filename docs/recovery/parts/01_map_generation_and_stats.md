# Part 1: Map Generation and Stats Recovery Guide

## Overview
This component monitors map generation state, detects seed changes, tracks if the map is generating/resetting, and reads the counts of interactable objects spawned on the map (like Shrines, Moais, Chests, etc.).

- **Target Files**:
  - Code: `src/game_data.py`
  - Unit Tests: `src/tests/test_game_data.py`

---

## Memory Chain Diagrams

### 1. Map Generation State
```
GameAssembly.dll + MAP_GENERATION_CONTROLLER_TYPE_INFO_OFFSET
  -> [Class Pointer]
    -> +0xB8 (CLASS_STATIC_FIELDS_OFFSET) -> [Static Fields Pointer]
      -> +0x10 (MAP_GENERATION_IS_GENERATING_OFFSET) -> bool (is_generating)
      -> +0x2C (MAP_GENERATION_MAP_SEED_OFFSET)      -> int (map_seed)
```

### 2. Map Controller State
```
GameAssembly.dll + MAP_CONTROLLER_TYPE_INFO_OFFSET
  -> [Class Pointer]
    -> +0xB8 (CLASS_STATIC_FIELDS_OFFSET) -> [Static Fields Pointer]
      -> +0x10 (MAP_CONTROLLER_CURRENT_MAP_OFFSET)   -> pointer (current_map_ptr)
      -> +0x18 (MAP_CONTROLLER_CURRENT_STAGE_OFFSET) -> pointer (current_stage_ptr)
      -> +0x21 (MAP_CONTROLLER_RESETING_OFFSET)      -> bool (is_resetting)
      -> +0x30 (MAP_CONTROLLER_RUN_CONFIG_OFFSET)    -> pointer (run_config_ptr)
```

### 3. Interactables Dictionary (Map Stats)
```
GameAssembly.dll + TYPE_INFO_OFFSET (Interactables dict static class)
  -> [Class Pointer]
    -> +0xB8 (CLASS_STATIC_FIELDS_OFFSET) -> [Static Fields Pointer]
      -> +0x0 (First field - dictionary pointer) -> [Dictionary Object]
        -> +0x18 (DICT_ENTRIES_OFFSET) -> [Entries Array Pointer]
        -> +0x20 (DICT_COUNT_OFFSET)   -> int (count of entries)

Each Dictionary Entry:
Entries Array + 0x20 (DICT_ENTRY_START) + (Index * 0x18 (DICT_ENTRY_SIZE))
  -> +0x8  (DICT_ENTRY_KEY_OFFSET)   -> [MonoString Pointer] (Stat name, e.g., "Greed Shrines")
  -> +0x10 (DICT_ENTRY_VALUE_OFFSET) -> [Container Object Pointer]
    -> +0x10 (CONTAINER_MAX_OFFSET)      -> int (max count)
    -> +0x14 (CONTAINER_CURRENT_OFFSET)  -> int (current count)
```

## Reversing Walkthrough (Cheat Engine & IL2CPP)

### 1. Locating Offsets using IL2CPP Dump
Search for these classes in the `dump.cs` file generated by IL2CPP Dumper:

- **`MapGenerationController`**:
  - Look for the static fields or metadata.
  - Locate `isGenerating` (offset `0x10` typically) and `mapSeed` (offset `0x2C`).
  - Note the class's `TypeInfo` address or symbol offset (e.g., `0x2F59000`).

- **`MapController`**:
  - Find fields like `currentMap`, `currentStage`, `isReseting` (note the single 's' typo in the game's code if applicable), and `runConfig`.
  - Note the class's `TypeInfo` address (e.g., `0x2F58E08`).

- **`InteractablesManager` / `MapStats`**:
  - Find where the dictionary of interactables is held.
  - Search for static fields of type `Dictionary<string, ...>` or `Dictionary<string, MapStatContainer>`.
  - Note the `TypeInfo` offset for the class holding the dictionary (e.g., `0x2FB5E68`).

### 2. Cheat Engine Live Verification
- **Verify Map Generation**:
  - Scan for 4-byte integers in Cheat Engine when restarting a map. The seed changes every run. Look for memory locations that change to a new random integer when the map generation begins.
  - Trace pointer pathways back to `GameAssembly.dll` base.

- **Verify Interactables Counts**:
  - Scan for integers representing known spawns (e.g., moais = 2, greed shrines = 1).
  - Walk the dictionary entries in memory. If a new game update changes the dictionary layout (e.g. .NET Dictionary structure shifts), verify the `DICT_ENTRIES_OFFSET` (`0x18`), `DICT_COUNT_OFFSET` (`0x20`), and `DICT_ENTRY_SIZE` (`0x18`).
