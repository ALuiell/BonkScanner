Описание интерфейса BonkScanner
Источник анализа: код `gui.py`, `config.py`, `config.json`, папка `media`
Дата фиксации: 2026-05-11

Этот документ нужен как текстовая спецификация текущего интерфейса, чтобы при переносе на PySide сохранить внешний вид и расположение элементов максимально близко к существующему варианту.

Важно:
- Ниже описан интерфейс именно по текущей ветке и текущему коду.
- В этой версии интерфейс в основном рисуется через Qt stylesheet, а не PNG-фонами.
- Скриншоты можно приложить к этому документу как визуальное дополнение.
- При переносе можно аккуратно улучшать визуальные решения, если это не вредит понятности и общему стилю.

==================================================
1. Общий стиль окна
==================================================

Главное окно:
- Заголовок окна: `BonkScanner v{CURRENT_VERSION}`.
- Размер окна при старте: `1280x760`.
- Минимальный размер окна: `1080x640`.
- Иконка окна: `media/bonkscanner_icon.ico`.

Общий визуальный стиль:
- Основной фон всего приложения: `#10141B`.
- Основной цвет текста: `#E5E7EB`.
- Базовый размер шрифта: `13px`.
- Интерфейс темный, сине-серый, без светлых панелей.

Стиль групповых блоков (`QGroupBox`):
- Фон: `#111827`.
- Рамка: `1px solid #243042`.
- Скругление: `8px`.
- Заголовок блока смещен влево, с небольшим внутренним отступом.

Стиль полей ввода и списков:
- Используется для `QLineEdit`, `QTextEdit`, `QPlainTextEdit`, `QListWidget`.
- Фон: `#0B1220`.
- Рамка: `1px solid #2B3648`.
- Скругление: `6px`.
- Внутренний отступ: `6px`.

Стиль обычных кнопок (`QPushButton`):
- Основной цвет: `#1F6AA5`.
- Цвет при наведении: `#2A80C0`.
- Цвет текста: белый.
- Без рамки.
- Скругление: `6px`.
- Внутренний отступ: `8px 12px`.

Стиль вкладок:
- Общая рамка блока вкладок: `1px solid #243042`.
- Скругление области вкладок: `8px`.
- Неактивная вкладка: фон `#172030`.
- Активная вкладка: фон `#22314A`.
- У вкладок скруглены верхние углы.

==================================================
2. Верхняя часть окна
==================================================

В верхней части окна расположен центрированный заголовок:
- Текст: `BonkScanner`.
- Стиль заголовка: объект `SectionHeader`.
- Размер шрифта: `20px`.
- Насыщенность: bold / `font-weight: 700`.

Под заголовком расположен центральный логотип:
- Для сохранения интерфейса при переносе нужно использовать файл: `media/bonkscanner_icon2.png`.
- Логотип выровнен по центру и расположен прямо под названием программы.
- Это именно крупный логотип в центре шапки, а не иконка окна.
- При переносе его лучше масштабировать пропорционально, сохраняя читаемость изображения.

Важно:
- `media/bonkscanner_icon.ico` нужно считать иконкой окна приложения.
- `media/bonkscanner_icon2.png` нужно считать основным логотипом в центре интерфейса.

==================================================
3. Основная компоновка окна
==================================================

Ниже шапки интерфейс делится на 2 колонки через горизонтальный `QSplitter`:
- Левая колонка уже и служит для выбора шаблонов / score-режима.
- Правая колонка шире и содержит логи, статистику, live stats и recordings.
- Стартовое соотношение ширин: примерно `360 / 900`.

Нижняя часть правой колонки содержит общую панель управления:
- Кнопка `Settings` слева.
- Текстовый статус по центру.
- Кнопка `Start (...)` или `Stop` справа.

==================================================
4. Левая колонка
==================================================

В левой колонке находится `QTabWidget` с 2 вкладками:
- `Templates`
- `Scores`

------------------------------------------
4.1. Вкладка Templates
------------------------------------------

Содержимое:
- Прокручиваемый список шаблонов.
- Внизу ряд из трех кнопок:
  `+ Add`
  `Edit`
  `Delete`

Все три кнопки используют стандартный стиль обычной кнопки:
- Цвет: `#1F6AA5`
- Hover: `#2A80C0`
- Белый текст

Каждый шаблон отображается как `QCheckBox`, стилизованный под карточку:
- Фон карточки: `#121A27`
- Hover-фон: `#162133`
- Рамка: `1px solid #2B3648`
- Левая цветная полоса: `4px solid {цвет шаблона}`
- Скругление: `8px`
- Внутренний отступ: `10px 12px`
- Текст жирный: `font-weight: 700`

Формат текста шаблона:
- `ИМЯ_ШАБЛОНА (условия)`
- Пример: `PERFECT (S+M: 8, Mic: 2, B: 2)`

Актуальные шаблоны и их цвета по текущему `config.json`:
- `LIGHT` -> цвет текста/акцента `#F8FAFC`
- `MERCHANT` -> `#22D3EE`
- `GOOD` -> `#22C55E`
- `PERFECT` -> `#FACC15`
- `PERFECT+` -> `#FB7185`
- `BOSS RUSH` -> `#EF4444`
- `BOSS RUSH+` -> `#E879F9`

Актуально активны сейчас:
- `MERCHANT`
- `PERFECT`
- `PERFECT+`

Визуально эта вкладка выглядит как темный список цветных карточек-чекбоксов с ярким акцентом слева у каждой строки.

------------------------------------------
4.2. Вкладка Scores
------------------------------------------

Содержимое:
- Верхний блок `Active Tiers`.
- Ниже большое read-only текстовое поле с описанием текущей score-системы.
- Внизу кнопка `Edit Settings`.

Кнопка `Edit Settings`:
- Обычная синяя кнопка.
- Цвет: `#1F6AA5`
- Hover: `#2A80C0`

Блок `Active Tiers`:
- Содержит чекбоксы:
  `Light`
  `Good`
  `Perfect`
  `Perfect+`

По текущему `config.json` активен только:
- `Perfect+`

Поле описания score-системы показывает:
- список активных tier-ов
- thresholds
- weights
- microwave multipliers

Это read-only темное текстовое поле на фоне `#0B1220`.

==================================================
5. Правая колонка
==================================================

В правой колонке находится `QTabWidget` с 4 вкладками:
- `Logs`
- `Session Stats`
- `Live Stats`
- `Recordings`

------------------------------------------
5.1. Вкладка Logs
------------------------------------------

Содержимое:
- Один большой `QTextEdit` только для чтения.
- Шрифт: `Consolas`, размер `11`.
- Используется как лог-консоль.

Текст в логах может быть цветным. Используются, например, такие цвета:
- `WHITE` -> `#F8FAFC`
- `CYAN` -> `#22D3EE`
- `GREEN` -> `#22C55E`
- `YELLOW` -> `#FACC15`
- `LIGHTRED_EX` -> `#FB7185`
- `RED` -> `#EF4444`
- `MAGENTA` -> `#E879F9`
- `BLUE` -> `#60A5FA`
- `LIGHTBLUE_EX` -> `#93C5FD`
- `DEFAULT` -> `#E5E7EB`

------------------------------------------
5.2. Вкладка Session Stats
------------------------------------------

Содержимое расположено в прокручиваемой области.

Текстовые строки:
- `Session Time: 00:00:00`
- `Session Rerolls: 0`
- `Total Rerolls: {значение из config}`
- `Rerolls per Minute (RPM): 0.0`
- `Best Map Found: None`
- `Worst Map Found: None`

Ниже идет подпись:
- `Average Rerolls per Target:`

Под этой подписью расположен отдельный контейнер для динамической статистики по шаблонам / целям.

Визуально вкладка выглядит как вертикальный список текстовых блоков на темном фоне.

------------------------------------------
5.3. Вкладка Live Stats
------------------------------------------

Структура сверху вниз:
- Статус-строка: `Waiting for game...`
- Горизонтальный ряд управления:
  кнопка `Start Recording (F8)`
  справа текст `Live stats`
- Ниже горизонтальный слайдер timeline
- Ниже подпись `Timeline: live stats`
- Ниже блок `Items`
- Ниже несколько блоков `Stats Group 1`, `Stats Group 2`, и т.д.

Кнопка записи live stats:
- В обычном состоянии:
  текст `Start Recording (F8)`
  фон `#1f538d`
  текст белый
- Во время записи:
  текст `Stop Recording`
  фон `#b30000`
  текст белый

Блок `Items`:
- Групповой блок с текстом списка предметов.
- Текст переносится по строкам.

Блоки статистики:
- Называются `Stats Group 1`, `Stats Group 2`, и т.д.
- Внутри используется `QFormLayout`: слева название стата, справа значение.
- Значение выровнено вправо.
- Минимальная ширина правой колонки значений: `72`.

Общее ощущение:
- Это техническая вкладка с живыми данными персонажа, с упором на список числовых параметров и timeline записи.

------------------------------------------
5.4. Вкладка Recordings
------------------------------------------

Компоновка делится на две части:
- Слева список записей (`QListWidget`)
- Справа подробности выбранной записи

Левая часть:
- Список сохраненных recording-файлов.
- Если записей нет, показывается строка `No saved recordings`.

Правая часть:
- Статус сверху: `Select a recording`
- Ниже строка:
  поле ввода имени
  кнопка `Rename`
  кнопка `Delete`
- Ниже timeline slider
- Ниже подпись времени `Timeline: --`
- Ниже блок `Items`
- Ниже прокручиваемая область с группами `Snapshot Group 1`, `Snapshot Group 2`, и т.д.

Кнопки `Rename` и `Delete`:
- Обычные синие кнопки
- Цвет: `#1F6AA5`
- Hover: `#2A80C0`

Визуально вкладка похожа на файловый менеджер записи слева плюс панель просмотра снимка справа.

==================================================
6. Нижняя панель управления
==================================================

Эта панель находится внизу правой колонки.

Элементы слева направо:
- Кнопка `Settings`
- Текст `Status: IDLE` / `Status: RUNNING` / `Status: ARMED` / `Status: WAITING FOR GAME`
- Кнопка `Start (F6)` или `Stop`

Кнопка `Settings`:
- При переносе ее нужно оформлять с использованием `media/settings_icon.png`.
- `settings_icon.png` нужно считать иконкой кнопки настроек.
- Текст `Settings` можно оставить рядом с иконкой.

Кнопка запуска:
- В неактивном состоянии текст `Start (F6)`
- В активном состоянии текст `Stop`
- Использует стандартный стиль кнопки:
  фон `#1F6AA5`
  hover `#2A80C0`

Статус:
- Отображается как обычный `QLabel`, без специальной заливки.

==================================================
7. Окна и диалоги
==================================================

------------------------------------------
7.1. Settings
------------------------------------------

Заголовок окна:
- `Settings`

Размер:
- примерно `420x320`

Поля:
- `Scan Hotkey`
- `Reset Hotkey`
- `Min Reroll Delay (s)`
- `Reset Hold Duration (s)`
- чекбокс `Use native hook restart`

Кнопки внизу:
- `Check for Updates`
- `Support`
- `Save`

Все кнопки обычные синие.

------------------------------------------
7.2. Scores Settings
------------------------------------------

Заголовок:
- `Scores Settings`

Размер:
- примерно `460x560`

Содержимое:
- блок `Active Tiers`
- блок `Thresholds`
- блок `Weights`
- блок `Microwave Multipliers`
- внизу кнопки:
  `Reset to Defaults`
  `Save Settings`

------------------------------------------
7.3. Add Template / Edit Template
------------------------------------------

Заголовок:
- `Add Template` или `Edit Template`

Размер:
- примерно `420x260`

Поля:
- `Template Name`
- `S+M Total (optional)`
- `Shady Guy (min)`
- `Moais (min)`
- `Microwaves (min)`
- `Boss Curses (min)`

Кнопка:
- `Save Template`

------------------------------------------
7.4. Delete Templates
------------------------------------------

Заголовок:
- `Delete Templates`

Размер:
- примерно `320x280`

Содержимое:
- текст `Select templates to delete:`
- список чекбоксов пользовательских шаблонов
- кнопка `Delete Selected`
- стандартная кнопка отмены `Cancel`

------------------------------------------
7.5. Delete Recording
------------------------------------------

Заголовок:
- `Delete Recording`

Содержимое:
- текст `Delete recording '{имя}'?`
- кнопки `Cancel` и `Delete`

------------------------------------------
7.6. Enable Native Hook Restart
------------------------------------------

Заголовок:
- `Enable Native Hook Restart`

Размер:
- примерно `420x220`

Содержимое:
- крупный заголовок `Enable native hook restart?`
- поясняющий текст про native helper и права администратора
- кнопки `Cancel` и `Enable`

==================================================
8. Привязка PNG / ICO ресурсов
==================================================

Ресурсы, которые нужно считать базовыми для переноса интерфейса:

1. `media/bonkscanner_icon.ico`
- Используется как иконка окна приложения.

2. `media/bonkscanner_icon2.png`
- Размер файла изображения: `560x443`.
- Его нужно использовать как центральный логотип в шапке интерфейса, под названием `BonkScanner`.
- Это отдельный визуальный элемент, не связанный с иконкой окна.

3. `media/settings_icon.png`
- Размер файла изображения: `800x800`.
- Его нужно использовать для кнопки `Settings`.
- Это иконка настроек, привязанная к кнопке настроек.

4. `media/button.png`
- Размер файла изображения: `2172x724`.
- Можно использовать как декоративную основу обычных кнопок, если это уместно в новой реализации.

5. `media/red_button.png`
- Размер файла изображения: `1536x1024`.
- Можно использовать как декоративную основу для кнопок активной записи, остановки или других важных акцентных действий.

==================================================
9. Что важно сохранить при переносе на PySide
==================================================

Если цель сохранить интерфейс "плюс-минус таким же", то лучше перенести следующие принципы:

- Темную синевато-серую тему с хорошей читаемостью.
- Верхнюю центрированную шапку: название `BonkScanner` и логотип под ним.
- Деление на 2 колонки: слева режимы/шаблоны, справа рабочие вкладки.
- Левую вкладку `Templates` как список крупных цветных чекбоксов-карточек.
- Правую часть с вкладками `Logs`, `Session Stats`, `Live Stats`, `Recordings`.
- Нижнюю панель управления с `Settings`, текстовым статусом и `Start/Stop`.
- Синюю базовую кнопку для большинства действий.
- Красную кнопку записи только в активном состоянии.

Если нужно сохранить не только структуру, но и "старую атмосферу" интерфейса, нужно использовать PNG так:
- `bonkscanner_icon2.png` как центральный логотип
- `settings_icon.png` как иконка кнопки настроек
- `button.png` как декоративный фон обычных кнопок
- `red_button.png` как декоративный фон кнопок стопа/записи

==================================================
10. Короткий вывод
==================================================

Текущий интерфейс BonkScanner должен восприниматься как:
- темный
- аккуратный
- технический по характеру
- построенный на вкладках
- визуально понятный и не перегруженный

При переносе главное сохранить:
- понятную структуру
- визуальную иерархию
- различимость элементов
- хороший контраст
- отсутствие сливающихся или слишком однообразных блоков

Основные ориентиры для переноса:
- документ `docs/interface_description_ru.txt`
- текущая структура интерфейса
- скриншоты как визуальный пример
