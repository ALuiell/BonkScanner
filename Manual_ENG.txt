BonkScanner - Quick Manual
==========================

What It Is
----------
BonkScanner is a desktop tool for automatic map rerolling in Megabonk.
You launch the app, choose a search mode, and it keeps rerolling until it finds
a map that matches your target.


What The Program Includes
-------------------------
- `Templates` mode
- `Scores` mode
- `Session stats`
- `Live Stats` recording and run inspection
- `Recordings` playback
- `OBS Overlay` browser-source output
- `Twitch Bot` Twitch chat integration
- stage summaries for live runs and recordings
- standard keyboard restart mode
- `Native hook` restart mode
- `Settings`
- live log output


How To Use It
-------------
1. Start the game.
2. Start BonkScanner.
3. Choose `Templates` or `Scores`.
4. Select the conditions you want.
5. Press `Start`.
6. Use the scan hotkey to arm the loop.
7. Wait for the target map.

When a map matches your conditions, the program stops and shows the result in
the log.


Templates Mode
--------------
This is the strict rule-based mode.

The map is checked against specific conditions, for example:
- `Shady Guy`
- `Moais`
- `Microwaves`
- `Boss Curses`
- `S+M`

Advantages:
- simple and easy to understand;
- good for targeted searching;
- useful when you know exactly what kind of map you want.

The `Templates` tab also includes an `Edit` button.
It opens the template manager, where you can:
- see all templates in one list;
- open a template card;
- change the values inline;
- save the template directly there.


Scores Mode
-----------
This mode evaluates the map with an overall scoring system.

Instead of using only strict template rules, the program looks at the general
value of the map and places it into a tier such as:
- `Light`
- `Good`
- `Perfect`
- `Perfect+`

Advantages:
- more flexible filtering;
- useful for finding generally strong maps;
- better when strong maps can have different stat combinations.


Templates vs Scores
-------------------
- `Templates` = strict conditions
- `Scores` = flexible overall evaluation

If you want clear and predictable matching, use `Templates`.
If you want broader and smarter filtering, use `Scores`.


Session Stats
-------------
`Session stats` show information about the current session.

They help you quickly see:
- how many rerolls happened;
- total rerolls saved by the app;
- how quickly target maps are found;
- how the current session is going overall.


Live Stats And Recordings
-------------------------
The app can record player stats snapshots while you play.

This is useful if you want to:
- review how a run developed over time;
- compare snapshots at different moments;
- keep a local recording of player stats, item lists, weapons, and stage progress.

The `Live Stats` and `Recordings` tabs can show:
- current player level and mob kills;
- passive items with rarity colors and a total item count;
- weapon levels and upgraded weapon stats;
- a `Stage Summary` table with time, kills, and item gains for stages 1-4;
- a `New Items` card that shows what changed since the previous snapshot.

Large kill counts are formatted with thousands separators, for example
`1,291,146`, so long runs are easier to read.

Recordings are designed to stay together across normal stage transitions. If a
map seed changes because the run moved to the next stage, the app should not
split the recording into a new file.

Passive item reading uses the normal passive inventory path first and can fall
back to the main `PlayerInventory.ItemInventory` path. This helps when items are
added by mods or external tools.

You can start recording from the `Live Stats` tab or by using the record hotkey.
The snapshot interval is configurable in `Settings`.


OBS Overlay
-----------
The `OBS Overlay` tab runs a local browser-source overlay for stream layouts.
It stays on the same PC and listens on `127.0.0.1`, so it is meant for OBS or a
normal browser on your own machine.

Turn on `OBS Overlay server`, then add the `Local OBS URL` to OBS as a Browser
Source. The URL includes a port, for example `17845`. A port is the local number
OBS uses to connect to BonkScanner's overlay server. If the port is already used
by another program, change it in the tab and update the OBS URL.

Visual Layout Editor:
You can open the overlay link in any browser with `?edit=true` (for example,
`http://127.0.0.1:17845/overlay?edit=true`). This opens the visual editor where
you can drag widgets around and configure your custom canvas size (HUD resolution).
All changes are saved instantly and automatically! When absolute positioning
is active, the overlay resizes to your chosen resolution (default: 1920x1080).
Make sure to set the identical Width and Height settings in OBS Browser Source.

Recording is not required for the overlay. It uses live stats reads. You can
choose visible widgets and configure tracked items by searching for an item,
choosing whether it should count on `Map 1 only`, and adding it to the list.
The overlay itself is transparent and designed to sit compactly over gameplay.


Twitch Bot
----------
The `Twitch Bot` tab runs a built-in Twitch IRC chat bot for your channel.
This allows chat viewers to query your active gameplay stats in real-time!

How to configure:
1. Go to the `Twitch Bot` tab.
2. Click `Connect to Twitch`. This opens a browser window for secure, official Twitch OAuth authorization.
3. Click authorize. Once completed, the status updates to `Connected as [YourUsername]`.
4. Configure the bot's settings (command access tier, cooldown delay, active commands, and automated announcements).
5. Click `Start Bot`.

Standard Mode And Native Hook
-----------------------------
Standard mode is the normal keyboard-based reroll mode.
It is the main option for most users.

`Native hook` is an alternate restart mode.
It can be enabled in `Settings`.

Advantages of `Native hook`:
- may feel better in some usage scenarios;
- may be more stable on some systems;
- can keep working more reliably while the user is alt-tabbed and not focused
  on the game window.

If standard mode already works well, you can stay on it.
If you want a better background-use option, try `Native hook`.


Settings
--------
`Settings` contains the main program options.

Important items:
- `Scan Hotkey`
  The hotkey used to arm or pause the main scan loop.

- `Reset Hotkey`
  The hotkey used by standard keyboard restart mode.

- `Record Hotkey`
  Starts or stops live player stats recording.

- `Min Reroll Delay (s)`
  The minimum delay between a reroll and the next check cycle.
  If this value is too low, game behavior or reading stability may suffer.

- `Reset Hold Duration (s)`
  How long the reroll key is held in standard keyboard restart mode.
  If it is too short, rerolls may sometimes fail.

- `Snapshot Interval (s)`
  How often the app saves a player stats snapshot while recording.

- `Use native hook restart`
  Enables `Native hook` mode.


Config
------
The program uses `config.json`.

This is the file where BonkScanner stores your settings, such as:
- selected mode;
- active templates;
- `Scores` settings;
- reroll settings;
- recording hotkey and snapshot interval;
- `Native hook` state.

You normally do not need to create this file yourself.
BonkScanner creates and updates `config.json` automatically.


Best Starting Point
-------------------
If you are new to the program:
- start with `Templates`;
- use standard mode first;
- then try `Scores`, live recordings, and `Native hook` if you want more flexibility.


Short Summary
-------------
- `Templates` = strict search
- `Scores` = flexible evaluation
- `Session stats` = current-session overview
- `Live Stats` = real-time player stats and recording
- `Stage Summary` = per-stage time, kills, and item gains
- `Twitch Bot` = built-in bot for live stats output and auto-announcements in chat
- `Native hook` = alternate restart mode that can work better in the background
- `config.json` = the file where the program stores your settings
